Animaciones y Scroll Reveal

Sistema de reveal via IntersectionObserver registrado en Layout.astro. Agregar clase base → transición a .visible al entrar en viewport.

↑
.reveal-up → .visible
Slide desde abajo + fade in. Efecto más común en tarjetas y secciones.
←
.reveal-left → .visible
Slide desde la izquierda. Para elementos que entran desde el lateral.
→
.reveal-right → .visible
Slide desde la derecha. Para sidebars y contenido alterno.
◎
.reveal-scale → .visible
Scale desde 0.9 + fade in. Para badges, iconos y elementos puntuales.
💡
Combinar con .delay-100 a .delay-700 para crear efecto stagger en grids de tarjetas. Ej: class="reveal-up delay-200"
Navbar scroll behavior
// En Layout.astro — JS inline
window.addEventListener('scroll', () => {
  if (window.scrollY > 60) {
    nav.classList.add('scrolled');
  } else {
    nav.classList.remove('scrolled');
  }
});

Componentes React

⚛️
Todos los componentes usan client:load en las páginas Astro. Se hidratan inmediatamente en el cliente.
TSX BookingForm.tsx
Formulario principal de reserva. Recolecta fechas, huéspedes y contacto. Guarda datos en localStorage y redirige a /checkout. Soporta código promo VERANO2026 (15% descuento).
  • itemId number ID del tour/hotel/transfer
  • itemType string 'hotel' | 'tour' | 'transfer'
  • basePrice number Precio base para cálculo
TSX CheckoutPage.tsx
Lee datos guardados en localStorage por BookingForm. Muestra resumen de reserva, permite seleccionar método de pago y confirmar la transacción.
  • (sin props) — Lee directamente de localStorage
TSX HotelList.tsx
Grid de tarjetas de hoteles con estilo .card-lift. Parsea amenities con try/catch (pueden venir como JSON string). Recibe hoteles ya filtrados del componente padre.
  • hotels Hotel[] Array de hoteles a renderizar
TSX HotelFilters.tsx
Componente de filtros para la página /hotels. Controla: búsqueda por texto, ciudad, estrellas, rango de precio y ordenamiento. Comunica estado filtrado hacia arriba via callback.
  • hotels Hotel[] Todos los hoteles (sin filtrar)
  • onFilter function Callback con hoteles filtrados

Capa de API

Base URL
import.meta.env.PUBLIC_API_URL || 'http://localhost:8000/api'

Hotels

GET /hotels api.getHotels()
GET /hotels?featured=1 api.getFeaturedHotels()
GET /hotels/{slug} api.getHotel(slug)

Tours

GET /tours api.getTours()
GET /tours?featured=1 api.getFeaturedTours()
GET /tours/{slug} api.getTour(slug)

Transfers

GET /transfers api.getTransfers()
GET /transfers?featured=1 api.getFeaturedTransfers()
GET /transfers/{slug} api.getTransfer(slug)

Banners

GET /banners?position={position} api.getBanners(position)
💡
El parámetro position en getBanners es opcional — default 'home'. Posibles valores: 'home', 'hotels', 'tours', etc.

Patrones y Gotchas

⚠️
Parseo de Arrays JSON desde API
Los campos amenities, features, included y not_included pueden llegar como JSON string o como array real dependiendo del endpoint. Siempre parsear con try/catch:
function parseArray(value: any): string[] {
  if (Array.isArray(value)) return value;
  try {
    return JSON.parse(value) || [];
  } catch {
    return value || [];
  }
}
🎟️
Código de Descuento
BookingForm.tsx implementa validación de código promocional. El código activo aplica un 15% de descuento al precio total.
// En BookingForm.tsx
const PROMO_CODE = "VERANO2026";
const PROMO_DISCOUNT = 0.15; // 15%

if (promoCode === PROMO_CODE) {
  total = total * (1 - PROMO_DISCOUNT);
}
🧭
Navbar Transparente en Homepage
La homepage pasa transparent={true} a Layout.astro, lo que agrega la clase .transparent-nav al navbar. Al hacer scroll mayor a 60px, JS agrega la clase .scrolled que aplica fondo opaco.
<!-- En index.astro -->
<Layout transparent={true}>
  ...
</Layout>

/* En global.css */
.transparent-nav { color: white; background: transparent; }
.transparent-nav.scrolled { background: white; color: navy; }
🔒
Páginas noindex
Las páginas /checkout y /bookings no deben ser indexadas por buscadores. Pasan noindex={true} al Layout para inyectar la meta tag correspondiente.
<!-- En checkout.astro y bookings.astro -->
<Layout noindex={true}>

<!-- Layout inyecta en <head>: -->
<meta name="robots" content="noindex, nofollow" />
🗺️
Sitemap Dinámico
/sitemap.xml es una API Route de Astro que fetcha todos los slugs del backend en tiempo de request y genera el XML dinámicamente. Configurado en public/robots.txt.
# public/robots.txt
User-agent: *
Allow: /
Disallow: /checkout
Disallow: /bookings
Sitemap: https://bookingcaribe.com/sitemap.xml
💾
Flujo de Reserva — localStorage
El flujo de reserva usa localStorage como canal entre páginas. BookingForm guarda, CheckoutPage lee. No hay state management global.
// BookingForm.tsx — guarda
localStorage.setItem('booking', JSON.stringify({
  itemId, itemType, dates, guests, contact, total
}));
router.push('/checkout');

// CheckoutPage.tsx — lee
const booking = JSON.parse(
  localStorage.getItem('booking') || '{}'
);
